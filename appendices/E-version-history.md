> **Note:** Save compatibility between versions is not guaranteed unless explicitly stated. Major version changes (e.g., v1.x to v2.x) typically break save compatibility. When updating Aurora, back up your save files and database before applying patches.

## E.2 Version Compatibility Guide

### Major Version Transitions

- **v1.x to v2.0.0**: Not save compatible. The v2.0 release introduced significant database schema changes requiring a fresh start.
- **v2.0.x to v2.1.x**: Check individual patch notes for compatibility status.
- **v2.1.0 and v2.1.1**: Explicitly save compatible.
- **v2.5.0 and v2.5.1**: Explicitly save compatible.

### Updating Between Minor Versions

Within a major version line (e.g., v2.2.0 to v2.3.0), saves may or may not be compatible. Always check the patch announcement thread before updating an active game.
